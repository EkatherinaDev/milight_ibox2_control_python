# Сценарии для Алисы и Салюта

Эта инструкция описывает, как настроить голосовые сценарии для управления MiLight iBox2
через Алису или Салют.

Важно: Алиса и Салют не могут напрямую запустить локальный `MiLightToggle.exe` на Windows.
Голосовой ассистент должен видеть свет как устройство умного дома. Для этого нужен один
из вариантов интеграции:

* готовая интеграция через Home Assistant или другой контроллер умного дома;
* приватный или публичный навык умного дома для Алисы с HTTPS-backend;
* интеграция Sber умного дома через Cloud-to-Cloud или MQTT-to-Cloud;
* локальный сервер-мост, который получает команду из облака и отправляет UDP-команду в iBox2.

После подключения MiLight как устройства умного дома используйте сценарии ниже.

## Рекомендуемые имена

Назовите устройство просто и понятно:

```text
Свет
```

или, если в доме несколько источников света:

```text
Свет MiLight
Свет в комнате
Лента
```

Для стабильной работы лучше сделать две отдельные команды: `включить` и `выключить`.
Команда `переключить` удобна, но зависит от сохраненного состояния и может ошибиться,
если свет включали с пульта или другого приложения.

## Алиса

Откройте приложение `Дом с Алисой` и создайте сценарии.

### Сценарий: включить свет

```text
Название: Свет MiLight включить

Если:
  Фраза: включи свет
  Фраза: включи свет в комнате
  Фраза: включи миллайт

Тогда:
  Устройство: Свет MiLight
  Действие: Включить
```

Команды голосом:

```text
Алиса, включи свет
Алиса, включи свет в комнате
Алиса, включи миллайт
```

### Сценарий: выключить свет

```text
Название: Свет MiLight выключить

Если:
  Фраза: выключи свет
  Фраза: выключи свет в комнате
  Фраза: выключи миллайт

Тогда:
  Устройство: Свет MiLight
  Действие: Выключить
```

Команды голосом:

```text
Алиса, выключи свет
Алиса, выключи свет в комнате
Алиса, выключи миллайт
```

### Сценарий: переключить свет

Используйте этот вариант только если backend работает как toggle, как текущий
`milight_gui.py` / `milight.py`.

```text
Название: Свет MiLight переключить

Если:
  Фраза: переключи свет
  Фраза: свет
  Фраза: нажми свет

Тогда:
  Действие: вызвать toggle-команду вашего backend
```

Команды голосом:

```text
Алиса, переключи свет
Алиса, свет
```

## Салют

Откройте приложение `Салют` или раздел умного дома в приложении Сбера и создайте
сценарии для подключенного устройства.

### Сценарий: включить свет

```text
Название: Свет MiLight включить

Условие запуска:
  Голосовая команда: включи свет
  Голосовая команда: включи свет в комнате
  Голосовая команда: включи миллайт

Действие:
  Устройство: Свет MiLight
  Состояние: Включено
```

Команды голосом:

```text
Салют, включи свет
Салют, включи свет в комнате
Салют, включи миллайт
```

### Сценарий: выключить свет

```text
Название: Свет MiLight выключить

Условие запуска:
  Голосовая команда: выключи свет
  Голосовая команда: выключи свет в комнате
  Голосовая команда: выключи миллайт

Действие:
  Устройство: Свет MiLight
  Состояние: Выключено
```

Команды голосом:

```text
Салют, выключи свет
Салют, выключи свет в комнате
Салют, выключи миллайт
```

### Сценарий: переключить свет

```text
Название: Свет MiLight переключить

Условие запуска:
  Голосовая команда: переключи свет
  Голосовая команда: свет

  Вызвать toggle-команду вашего backend
```

Команды голосом:

```text
Салют, переключи свет
Салют, свет
```

## Техническая схема интеграции

Минимальная рабочая схема выглядит так:

```text
Алиса / Салют
    |
    | голосовая команда
    v
Платформа умного дома
    |
    | HTTPS / Cloud-to-Cloud / MQTT-to-Cloud
    v
Ваш backend или контроллер
    |
    | UDP 5987 в локальной сети
    v
MiLight iBox2
    |
    v
Лампа / зона 0 (все зоны)
```

Для Алисы сценарии можно запускать голосом по фразе, по расписанию или по состоянию
другого устройства. Если делать собственный навык умного дома, backend должен быть
доступен по HTTPS и отвечать быстро.

Для Салюта устройство нужно добавить в умный дом Sber через поддерживаемую интеграцию.
Официальные варианты для разработчиков: Cloud-to-Cloud и MQTT-to-Cloud.

## Что лучше доработать в проекте

Сейчас GUI и консольный скрипт работают как переключатель: один запуск меняет состояние
на противоположное. Для голосовых ассистентов надежнее добавить отдельные команды:

```text
milight.py on
milight.py off
milight.py toggle
```

Тогда сценарии `включить свет` и `выключить свет` не будут зависеть от локально
запомненного состояния.
